# IMS.md

## 운영 기준 (오세영님 요청)

1. 모든 업무 브리핑/정리는 이 문서를 기준으로 따른다.
2. 마스터 데이터는 옵시디언(`obsidian-vault/`)이다.
3. 업무 체계는 `월간 → 주간 → 데일리` 순서로 운영한다.
4. 담당자 미지정 업무도 먼저 기록하고, 이후 배정한다.
5. 기본 응답은 한국어 존댓말로 한다.

## 현재 표준 구조
- 홈: `obsidian-vault/01_홈.md`
- 월간: `obsidian-vault/01_월간/`
- 주간: `obsidian-vault/02_주간/`
- 데일리: `obsidian-vault/03_데일리/`
- 팀원 할일: `obsidian-vault/02_할일/`
- 자산(업체/팀원): `obsidian-vault/03_자산/`

## 브리핑 원칙
- 먼저 전체 우선순위(월간/주간)
- 다음 오늘 실행 항목(데일리)
- 마지막으로 누락/리스크(미지정, 지연)
